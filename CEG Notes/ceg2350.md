### Disk and Devices
Disk have partitions, can have multiple partitions per disk \
There are block and character devices \
`lsblk` will list out the block devices and their partitions \
`fdisk` utility to modify partition tables specifically MBR partitions \
`gdisk` will allow to modify GPT partitions \
`gdisk`, `fdisk`, and `parted` can view and manage partition tables in Linux systems \
MBR - master boot record 
- Stored in first sector (512 bytes) of a disk
- Support disks sizes less than 2 TB
- Common to BIOS based motherboards
- Supports a maximum of 4 primary partitions

GPT - GUID Partition Table 
- Separately stores the bootloader on an EFI partition.  Redundantly stores the partition table itself
- Supports disk sizes greater than 2 TB
- Common to UEFI based motherboards
- Supports up to 128 partitions	

Windows -> Disk Management -> to see how many disks you have \
Every disk is gonna have some space taken up, those disks hold information for the partition table \
Partition is how we define a boundary on the disk \
Partition then decide what file system you want on it \
POST stands for Power On Self Test \
Correct order of the boot process steps in a typical PC -> BIOS/UEFI initialization, POST, bootloader, kernel initialization \
Type of hypervisor runs directly on the physical hardware without the need for a host operating system -> Type 1 \
BIOS:
- Limited security features.  Can use a password to prevent unauthorized access
- Supports MBR partitioning scheme
- Typically has a text-based interface with limited options and requires navigation using keyboard shortcuts. \
UEFO:
- Supports GPT partitioning scheme
- Offers secure boot to validate bootloader and driver signatures
- Offers a graphical user interface (GUI) with mouse support, making it more user-friendly and easier to navigate.

Windows uses NTFS filesystem \
MacOS uses HSF+ \
Linux uses ext4 \
An inode is an index node \
Stat shows you the information in an inode \
Inodes store:
- File size
- Last file access & file creation time
- File pointers to data blocks on the disk where the actual file contents are stored
- File permissions
- File links (hard links) pointing to the inode

Gateway \
